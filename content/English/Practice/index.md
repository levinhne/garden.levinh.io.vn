# Practice Exercises

## ✍️ Writing Practice

### Daily Tasks
- [ ] Write git commit messages in English
- [ ] Document code in English
- [ ] Write pull request descriptions
- [ ] Comment on GitHub issues
- [ ] Write technical blog posts (weekly)

### Writing Templates

#### Pull Request Description
```markdown
## Summary
Brief description of changes

## Changes
- What changed and why
- Technical approach

## Testing
How the changes were tested

## Screenshots (if applicable)
Visual proof of changes
```

#### Code Documentation
```go
// CalculateDiscount computes the discount amount based on the user's tier.
// It returns 0 if the user is not eligible for any discount.
//
// Parameters:
//   - user: The user object containing tier information
//   - amount: The original purchase amount
//
// Returns:
//   - float64: The discount amount to be applied
//   - error: Error if the calculation fails
```

## 🗣️ Speaking Practice

### Daily Routine
- [ ] Explain code changes out loud
- [ ] Practice technical presentations (5 min)
- [ ] Record yourself explaining concepts
- [ ] Shadow technical YouTube videos

### Topics to Practice
1. Explain your current project
2. Describe a technical challenge you solved
3. Present a system design
4. Walkthrough your code
5. Explain a design pattern

## 👂 Listening Practice

### Comprehension Exercises
- [ ] Watch tech talk without subtitles
- [ ] Take notes during podcasts
- [ ] Summarize what you heard
- [ ] Follow along with conference talks

### Resources
- Conference talks (15-20 min)
- Technical podcasts (30 min)
- Code review videos
- System design interviews

## 📖 Reading Practice

### Daily Reading
- [ ] Read 1 technical article
- [ ] Browse GitHub trending repos
- [ ] Read API documentation
- [ ] Follow engineering blogs

### Comprehension Strategy
1. **Skim**: Get the main idea
2. **Scan**: Find specific information
3. **Read carefully**: Understand details
4. **Summarize**: Write key points

## 🎯 Weekly Goals

### Week 1-4: Foundation
- Learn 50 technical terms
- Write 5 git commits in English
- Watch 2 conference talks
- Read 5 technical articles

### Week 5-8: Application
- Document 3 functions in English
- Write 2 pull request descriptions
- Practice 1 technical presentation
- Participate in 1 English discussion

### Week 9-12: Mastery
- Write 1 technical blog post
- Give 1 code walkthrough in English
- Review 5 PRs with English comments
- Explain 3 system designs
